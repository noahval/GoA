**Goal**: Define standard UI elements, variables, and patterns for all Level 1 (Copper Era) scenes

**Success Criteria**:
- All Level 1 scenes follow consistent UI patterns

**Prerequisites**:

- Phase 1.3 (Global Autoload) - Global.gd exists
- Level1Vars autoload - level_1_vars.gd exists

---

## Overview

This plan defines the **content standards** for Level 1 (Copper Era) scenes. While 1.x-scene-template.md defines the universal node structure (Background, MainLayout, PlayArea, Menu, NotificationBar), this plan specifies what goes **inside** those containers for Level 1.

### Key Design Principles

- **Consistency**: All Level 1 scenes follow the same UI patterns
- **Standard elements**: Currency display, navigation, stats appear the same everywhere
- **Scene flexibility**: Scenes can add custom content while maintaining standards
- **Inherits from universal template**: Builds on 1.x-scene-template.md structure

## Standard Menu Content (Right Side - 33%)

All Level 1 scenes should include these elements in `MainLayout/Menu`:

### 1. Currency Display Panel

**Node structure**:
```
Menu (VBoxContainer)
├── CurrencyPanel (PanelContainer)
│   └── CurrencyLabel (Label)
```

**Display format**:
```gdscript
$MainLayout/Menu/CurrencyPanel/CurrencyLabel.text = "Copper: %d" % Global.copper_current
```

**Update trigger**:
- On _ready()
- When Global.copper_current changes (connect to signal if available)

### 2. Navigation Buttons

**Pattern**: VBoxContainer with buttons for scene transitions

**Example structure**:
```
Menu (VBoxContainer)
├── [CurrencyPanel]
├── NavigationButtons (VBoxContainer)
│   ├── FurnaceButton (Button)
│   ├── BarButton (Button)
│   ├── ShopButton (Button)
```

**Button behavior**:
```gdscript
func _on_furnace_button_pressed():
    Global.change_scene("res://level1/furnace.tscn")
```

**Common scene paths** (Level 1):
- Furnace: `res://level1/furnace.tscn`
- Bar: `res://level1/bar.tscn`
- Shop: `res://level1/shop.tscn`
- Loading Screen: `res://level1/loading_screen.tscn`

## Standard PlayArea Content (Left Side - 66%)

The PlayArea is scene-specific but should follow these patterns:

### Interactive Elements

**Pattern**: Clickable objects that trigger actions

**Example** (furnace scene):

## Creating a New Level 1 Scene

**Step-by-step process**:

1. **Inherit from template**:
   - Right-click `level1/scene_template.tscn` → "New Inherited Scene"
   - Rename root node to scene name (e.g., "Forge", "Mine")

2. **Add standard menu content**:
   - Add CurrencyPanel with CurrencyLabel
   - Add NavigationButtons container with scene buttons
   - Add any other standard Level 1 menu elements

3. **Add scene-specific PlayArea content**:
   - Add interactive elements
   - Add visual feedback elements

4. **Attach script**:
   - Use standard script pattern above
   - Call ResponsiveLayout.apply_to_scene(self) in _ready()
   - Implement setup_ui() and connect_signals()

5. **Save scene**:
   - Save to `res://level1/[scene_name].tscn`

## Notes & Decisions

**Decision**: Level-specific content plan separate from universal template
- **Rationale**: Level 2 will have different UI needs, different currency, different mechanics
- **Benefit**: Universal template stays clean, each level defines its own standards
- **Scalability**: Level 2 gets its own `3.x-level2-scene-content.md`

**Decision**: Define standards incrementally
- **Rationale**: You're building Level 1 scenes now, standards emerge from actual needs
- **Approach**: Mark sections [TO BE DEFINED] and fill in as you build
- **Benefit**: Practical standards based on real implementation, not theoretical

**Decision**: Standard menu content mandatory, PlayArea content flexible
- **Rationale**: Menu provides navigation/info (should be consistent), PlayArea is gameplay (varies by scene)
- **Benefit**: Consistent user experience for navigation, creative freedom for gameplay
