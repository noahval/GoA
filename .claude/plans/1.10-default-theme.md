# Phase 1.10: Default Theme

**Goal**: Create UI theme resource ([default_theme.tres](../default_theme.tres)) for consistent styling across all scenes

**Success Criteria**:
- default_theme.tres created and configured
- Base font sizes set for 1280x720 resolution
- Color scheme defined
- All UI controls inherit theme styles automatically
- Theme scales correctly with responsive layout system

**Prerequisites**:
- Phase 1.1 (Project Setup) - Project and resolution configured

---

## Overview

Establishes the visual foundation for all UI elements in GoA. The default theme provides consistent styling, colors, and typography across all scenes, ensuring visual cohesion and reducing per-scene styling work.

### Key Design Principles

- **Consistency**: All UI elements use the same base styling
- **Scalable**: Font sizes designed for 1280x720 base, scale up for larger resolutions
- **Dark Theme**: Fits grimdark industrial aesthetic
- **Readable**: High contrast for text visibility
- **Minimal**: Simple, functional design without distractions

## Implementation Tasks

### 1. Create Theme Resource

**Steps:**
1. In Godot Editor: FileSystem → Right-click root
2. Create → Resource → Theme
3. Save as: `res://default_theme.tres`

### 2. Configure Default Font

**Font Selection**:
- Use built-in Godot font OR
- Import custom font (e.g., Roboto, Inter, or similar sans-serif)

**Base Font Sizes** (for 1280x720):
```gdscript
# In default_theme.tres Font Size settings
default_font_size = 16       # Body text
```

**Font Overrides for Specific Controls**:
- Button: 18px
- Label (headings): 24px
- Label (small/notification): 14px
- RichTextLabel: 16px

### 3. Define Color Scheme

**Base Colors** (grimdark industrial theme):

```gdscript
# Background colors
panel_bg = Color(0.15, 0.15, 0.15, 0.95)     # Dark gray panels
button_bg = Color(0.2, 0.2, 0.2, 1.0)        # Slightly lighter for buttons
button_hover = Color(0.3, 0.3, 0.3, 1.0)     # Hover state
button_pressed = Color(0.1, 0.1, 0.1, 1.0)   # Pressed state

# Text colors
text_normal = Color(0.9, 0.9, 0.9, 1.0)      # Light gray text
text_disabled = Color(0.5, 0.5, 0.5, 1.0)    # Dimmed disabled text
text_highlight = Color(1.0, 0.85, 0.4, 1.0)  # Gold highlight (important info)

# Accent colors
copper_accent = Color(0.72, 0.45, 0.2, 1.0)  # Copper/orange
silver_accent = Color(0.75, 0.75, 0.75, 1.0) # Silver
gold_accent = Color(1.0, 0.84, 0.0, 1.0)     # Gold

# Status colors
stat_gain = Color(0.4, 0.8, 0.4, 1.0)        # Green (positive)
danger = Color(0.9, 0.3, 0.3, 1.0)           # Red (negative/warning)
```

### 4. Style Common Controls

**Panel StyleBox**:
- Background: `panel_bg`
- Border: None or subtle 1px dark border
- Corner Radius: 4px (slight rounding)

**Button StyleBox**:
- Normal: `button_bg`
- Hover: `button_hover`
- Pressed: `button_pressed`
- Disabled: `button_bg` with 50% alpha
- Font Color: `text_normal`
- Border: 1px darker outline

**Label**:
- Font Color: `text_normal`
- Shadow: Optional subtle drop shadow for readability

**ProgressBar**:
- Background: Dark gray (`Color(0.1, 0.1, 0.1, 0.8)`)
- Fill: Accent color (varies by context - copper/silver/gold)
- Border: 1px outline

**LineEdit / TextEdit**:
- Background: Darker panel (`Color(0.1, 0.1, 0.1, 0.9)`)
- Font Color: `text_normal`
- Caret: `text_highlight`
- Border: 1px subtle outline

## Design Values (Reference)

### Font Sizes (1280x720 base)

| Element | Size | Purpose |
|---------|------|---------|
| Body Text | 16px | Default readable size |
| Buttons | 18px | Slightly larger for clarity |
| Headings | 24px | Section headers |
| Small/Notifications | 14px | Secondary info |

**Scaling**: These sizes are for 1280x720. The responsive layout system (Phase 1.11) will scale them proportionally for larger resolutions.

### Color Palette

**Base Theme**: Dark industrial grimdark
- **Backgrounds**: Dark grays (15-20% brightness)
- **Text**: Light gray (90% brightness)
- **Accents**: Copper/Silver/Gold (currency colors)
- **Status**: Green (gains), Red (warnings)

**Design Rationale**:
- High contrast ensures readability
- Dark theme reduces eye strain
- Metallic accents reinforce industrial setting
- Consistent color language (copper = low tier, gold = high tier)

## Notes & Decisions

**Decision 1**: Use built-in theme system instead of manual styling
- **Rationale**: Godot's theme system provides inheritance, consistency, and reduces code duplication
- **Benefit**: Change theme once, all scenes update automatically

**Decision 2**: Dark theme color scheme
- **Rationale**: Fits grimdark industrial setting, reduces eye strain for long play sessions
- **Aesthetic**: Reinforces oppressive labor atmosphere

**Decision 3**: Base font sizes for 1280x720
- **Rationale**: Designed for base resolution, responsive layout scales them up
- **Approach**: Define base once, scale algorithmically

**Decision 4**: Minimal decoration
- **Rationale**: Idle/incremental games benefit from clear information display over visual flourishes
- **Focus**: Readability and information density

---

## Integration Notes

**How Theme Interacts with Responsive Layout**:

1. **Theme defines base sizes** (for 1280x720)
2. **Responsive Layout system scales** (Phase 1.11) fonts and elements based on window size
3. **Scenes inherit theme automatically** via project settings

**Example**:
- Theme sets button font to 18px (base)
- Responsive Layout detects 1920x1080 window (150% scale)
- Button font becomes 27px (18 × 1.5)

This separation keeps the theme simple while allowing dynamic scaling.
